# 异常预警与信用精细化答辩材料

## 1. 创新点摘要

- 密码安全从单一 MD5 升级为 `BCrypt` 新写入，同时保留历史 MD5 登录兼容，保证存量用户无感迁移。
- 异常预警由“单条规则”升级为 MVP 规则引擎：
  - `CARE_INACTIVE`：重点人群连续未登录预警。
  - `DEMAND_SURGE`：社区 24 小时求助量骤增预警。
- 志愿服务闭环新增信用精细化：
  - 信用明细账本 `volunteer_credit_ledger`
  - 信用快照 `volunteer_credit_snapshot`
  - 看板展示累计信用分、近 30 天评分与履约率。

## 2. 阈值口径（可配置）

- 重点人群未登录天数：`careInactivityDays`，默认 `3`。
- 24 小时求助骤增最低阈值：`surge24hMinRequests`，默认 `5`。
- 求助骤增倍数阈值：`surgeMultiplier`，默认 `2.0`。

触发规则：

1. `CARE_INACTIVE`：`last_login_at < now - careInactivityDays`
2. `DEMAND_SURGE`：`current24h >= max(surge24hMinRequests, baseline7d * surgeMultiplier)`

去重策略：

- 每条规则按业务主键+日期生成 `dedup_key`，同一天仅入库一次并仅通知一次。

## 3. 信用分计算口径（V1）

- 基础分：`hours * 10`
- 评分加权：
  - 5 星：`+10%`
  - 4 星：`+5%`
  - 3 星：`0`
  - 2 星：`-5%`
  - 1 星：`-10%`
- 时效扣减：
  - 未超时：`1.00`
  - 超时 <=24h：`0.95`
  - 超时 >24h：`0.90`
- 单笔上限保护：`500`

## 4. 演示步骤（答辩可复现）

1. 在系统配置页修改告警阈值并保存。
2. 构造重点人群用户（`identity_tag` 含“孤寡老人”）且模拟连续未登录，等待任务触发。
3. 在近 24h 批量新增同社区需求数据，触发骤增预警。
4. 打开“服务过程监控”页面，查看新增字段：
   - 触发规则
   - 建议动作
5. 完成一笔服务核销后，查询信用账本与信用快照变更。
