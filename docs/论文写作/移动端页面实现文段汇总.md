# 论文移动端页面实现文段汇总

## 4.4.1.1 登录/注册页实现

### （1）页面布局与实现
登录页与注册页是移动端用户进入系统的第一入口，主要完成身份认证、账号创建和初始信息采集。两页均采用卡片化布局和分步引导设计，目标是降低首次使用门槛并提升表单填写效率。  

登录页功能如下：  
- 品牌与欢迎区。  
- 账号密码输入区。  
- 忘记密码与去注册入口。  
- 第三方快捷登录入口。  
- 协议提示区。  

注册页功能如下：  
- 身份选择（居民/志愿者）。  
- 注册方式切换（手机/邮箱）。  
- 基础信息填写（账号、姓名、联系方式、验证码、密码、地址）。  
- 志愿者技能填写。  
- 提交注册与跳转登录。  

### （2）关键技术与代码（保留2点）
- **登录校验与提交**：登录页使用表单校验后调用登录接口，成功即跳转。  
```ts
const onSubmit = form.handleSubmit((values) => {
  appAuthStore.login(values).then(() => {
    router.replace(redirect.value)
  })
})
```

- **注册后自动登录闭环**：注册成功后自动登录并进入用户首页。  
```ts
const res = await apiApp.register({ ...payload })
await appAuthStore.login({ account: form.username.trim(), password: form.password })
router.replace('/user')
```

## 4.4.1.2 用户首页实现

### （1）页面布局与实现
用户首页采用“顶部信息栏 + 个人摘要卡片 + 勋章区 + 快捷操作区 + 退出按钮”的结构。  
用户可查看社区绑定信息、个人积分与互助统计，并完成消息跳转、资料编辑、任务入口跳转等操作。  

### （2）关键技术与代码（保留2点）
- **首页数据加载**：页面挂载后拉取用户信息和看板统计。  
```ts
async function loadProfile() {
  await appAuthStore.hydrateUser()
  const res = await getUserDashboardSummary()
  if (res.code === 200 && res.data) dashboard.value = res.data
}
onMounted(loadProfile)
```

- **勋章展示交互**：支持“仅看已获得/查看全部”切换与详情弹窗。  
```ts
const visibleMedals = computed(() => showAllMedals.value ? medals : medals.filter(m => m.acquired))
function onOpenMedal(medal) {
  selectedMedal.value = medal
  showMedalModal.value = true
}
```

## 4.4.1.4 需求发布页实现

### （1）页面布局与实现
需求发布页采用“顶部导航 + 分组表单 + 提交按钮”结构。  
用户可选择服务类型、紧急程度，填写地址与联系人信息，补充需求描述后提交发布。  

### （2）关键技术与代码（保留2点）
- **动态描述模板**：根据服务类型自动生成描述文本。  
```ts
watch(() => form.serviceType, (nextType) => {
  if (form.description.trim()) return
  form.description = getDescriptionTemplate(nextType)
}, { immediate: true })
```

- **发布校验与提交**：校验必填项后调用发布接口。  
```ts
if (!form.serviceAddress.trim()) return toast.error('请填写服务地址')
if (!form.emergencyContactName.trim()) return toast.error('请填写联系人姓名')
const res = await createServiceRequest({ ...formPayload })
```

## 评价反馈页实现

### （1）页面布局与实现
评价反馈页采用“评分 + 补充说明 + 提交”结构。  
用户可选择 1-5 分并填写简短意见，提交后返回评价列表。  

### （2）关键技术与代码（保留1点）
- **评价提交与跳转**：  
```ts
const res = await api.post('/service-evaluation', {
  claimId: claimId.value,
  rating: Number(rating.value),
  content: content.value.trim(),
})
if (res.code !== 200) throw new Error(res.message || '提交失败')
router.replace({ path: '/hall-overview', query: { kind: 'reviews' } })
```
